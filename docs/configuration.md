# Configuration

Every knob in one place. Everything in `~/.opengriffin/.env` (or the project root `.env`).

## Provider selection

```bash
OPENGRIFFIN_PROVIDER=claude   # claude | anthropic | openai | gemini | deepseek | ...
OPENGRIFFIN_MODEL=            # optional — override the provider's default model
```

See [providers.md](providers.md) for the 21 supported providers and their default models.

## Provider keys

Set whichever you'll use. Keys are loaded into env at startup; the bot never sends them to opengriffin.com or any third party except the provider whose key it is.

```bash

OPENAI_API_KEY=
GEMINI_API_KEY=             # or GOOGLE_API_KEY
DEEPSEEK_API_KEY=
OPENROUTER_API_KEY=
XAI_API_KEY=
MISTRAL_API_KEY=
COHERE_API_KEY=
PERPLEXITY_API_KEY=
TOGETHER_API_KEY=
GROQ_API_KEY=
FIREWORKS_API_KEY=
CEREBRAS_API_KEY=
NVIDIA_API_KEY=
HUGGINGFACE_API_KEY=
LAMBDA_API_KEY=
NOVITA_API_KEY=

# Azure
AZURE_OPENAI_API_KEY=
AZURE_OPENAI_ENDPOINT=

# AWS Bedrock
AWS_ACCESS_KEY_ID=
AWS_SECRET_ACCESS_KEY=
AWS_REGION=us-east-1

# Ollama (local)

CUSTOM_BASE_URL=
```

If you set `OPENGRIFFIN_PROVIDER=claude` and have Claude Max OAuth via the Claude Code CLI, no key is needed — the SDK reads `~/.claude/.credentials.json`.

## Telegram (default gateway)

```bash
TELEGRAM_BOT_TOKEN=                    # from @BotFather
TELEGRAM_ALLOWED_USERS=                # comma-separated user ids; empty = open
TELEGRAM_HOME_CHANNEL=                 # default chat for cron deliveries / proactive messages
```

## Per-chat overrides

Some settings are per-chat, not global. Stored in `~/.opengriffin/aliases.json`:

| Setting | Telegram command | What |
|---|---|---|
| Per-chat sysprompt | `/sysprompt <text>` | Appended to the system prompt for THIS chat only |
| Per-chat provider | `/model <provider>` | Routes this chat's queries to a different provider |
| Per-chat model | `/model <provider> <model>` | Specific model on the per-chat provider |
| Quick-command alias | `/alias name=template` | Shortcut: `/run name [args]` expands and sends |

These don't affect cron jobs (which use the global config) or other chats.

## Hot-reloading

Most config changes require a bot restart. Exceptions:

- **MEMORY.md / USER.md / SOUL.md / CONSTRAINTS.md** — picked up on the next session start (no restart needed)
- **Per-chat sysprompt / model / aliases** — applied immediately
- **Skills under `~/.claude/skills/`** — picked up on next session start
- **`jobs.json`, `triggers.json`, `webhooks.json`** — restart required (the scheduler reads them at startup)

To restart on macOS launchd:
```bash
launchctl kickstart -k gui/$(id -u)/ai.opengriffin.bot
```

To restart on Linux systemd:
```bash
sudo systemctl restart opengriffin
```

## Verifying your config

```bash
opengriffin doctor
```

Outputs a table of:
- Selected provider + load status
- Whether `TELEGRAM_BOT_TOKEN` is set
- Whether the provider's required env vars are set
- Whether bundled skills are installed
- Whether the bot is currently running

If something's missing, the table tells you precisely what.
